# TulparHub — что сделано, передача разработчикам

## 1. Что это

B2B/B2C маркетплейс запчастей для грузовой и спецтехники в Казахстане (КАМАЗ, MAN, DAF, Volvo, Scania, HOWO, Shacman и др.). Прототип/MVP собран с помощью Claude (AI), задеплоен и работает.

- **Сайт (живой, открыт сейчас):** https://tulparhub-psi.vercel.app
- **Репозиторий:** https://github.com/tulparhubkz/tulparhub
- **34 коммита**, велась полная история разработки — каждый коммит можно открыть и посмотреть diff

## 2. Стек

| Слой | Технология |
|---|---|
| Frontend | Next.js 14 (App Router), TypeScript, React |
| Стили | Чистый CSS (без Tailwind), переменные в `globals.css` |
| База данных | Supabase (PostgreSQL) |
| Хостинг | Vercel (автодеплой из GitHub `main`) |
| Стейт на клиенте | Zustand + localStorage (корзина, избранное, гараж) |

## 3. Что уже работает

**Каталог**
- 20 279 деталей в базе (импортированы из CSV реального прайса)
- Фильтры: бренд, тип (OEM/аналог), наличие, цена, категория (система)
- Поиск по артикулу, названию, **по VIN-коду** (автоопределение марки/модели техники), по модели техники
- Карточка товара: фото, наличие по складам, цена розница/опт, B2B-режим, кросс-номера, аналоги
- Фото деталей — подтянуты с реальных каталогов запчастей (truckdrive.ru, autodoc, exist и др.), не стоковые фото

**Корзина и заказ**
- Добавление в корзину, попап "Добавлено в корзину" (как на autopiter.kz) с управлением количеством
- Страница корзины

**Избранное**
- Сохранение товаров в избранное (♡), отдельная страница `/wishlist`

**Мой гараж** (как на autopiter.kz)
- Добавление техники по VIN-коду
- **Декодер VIN** — по WMI-коду (первые 3 символа VIN) определяет марку/модель грузовика: MAN, DAF, Volvo, Scania, Mercedes-Benz, IVECO, Renault Trucks, КАМАЗ, МАЗ, HOWO, Shacman, FAW, Foton, DongFeng. Затем подбирает запчасти именно под эту модель
- Хранится в Supabase (таблица `garage`) + localStorage

**Помощь в подборе** (`/vin`)
- Форма "Подбор по VIN" — с автоопределением техники при вводе
- Форма "Подбор по параметрам" (год/марка/модель/двигатель)

**Шапка сайта**
- Выбор города (все 30 крупных городов Казахстана, с поиском)
- Многоязычность RU/KZ/EN (переключатель есть, переводов пока нет)
- Логотип TulparHub (заменён с ошибочного — был случайно загружен логотип университета СДУ)

## 4. Чего НЕТ (для разработчиков — куда расти)

- ⚠️ **Нет реальной авторизации** — страница `/auth` это макет (UI есть, логики работы с пользователями нет)
- ⚠️ **Нет оплаты** — нет интеграции с платёжным провайдером (Kaspi, банковские карты и т.п.)
- ⚠️ **Нет админ-панели** — товары добавлены напрямую в Supabase через скрипт, нет интерфейса управления каталогом
- ⚠️ **VIN-декодер приближённый** — определяет марку/модель по таблице WMI-кодов, это не полноценная интеграция с TecDoc/Laximo (платные базы данных автозапчастей с точным подбором по VIN). Для production-уровня точности нужна такая интеграция
- ⚠️ **Переводы KZ/EN** — кнопки есть, перевода текста интерфейса нет
- ⚠️ **Email/SMS уведомления** — не подключены
- ⚠️ **Реальные фото деталей по всем 20 279 товарам** — пока есть индивидуальные фото только для первых 500 (по OEM-номеру), остальные показывают фото по категории детали (общее фото "фильтр масляный" вместо фото конкретной детали)

## 5. Как показать разработчикам код и переписку

**Код:**
- Просто дать ссылку на репозиторий: https://github.com/tulparhubkz/tulparhub
- Там видна вся история — `git log`, каждый файл, каждый коммит с описанием на английском

**Переписка с Claude (весь процесс разработки):**
- Если разработчики используют Claude Code / Claude.ai — можно экспортировать историю сессии
- Либо просто переслать им скриншоты диалога — там видно по шагам что просилось и что менялось
- В каждом git-коммите есть описание (`git log`) — оно фактически дублирует "что было сделано" из переписки, так что код + git log уже достаточно прозрачен
